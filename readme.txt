Project: Memo App

Description:
-------------
A simple web-based memo application that allows users to log in with an access token, view categories, and manage memos (create, edit, delete) within each category.

Project Structure:
------------------
index.html                - (empty, not used)
style.css                 - (empty, not used)
lib/
    wait_load.js          - Utility for waiting until DOM is loaded (used in tests)
public/
    index.html            - Main HTML file for the app UI
    index.js              - Main JavaScript logic for the app
    style.css             - Main CSS styles for the app
test/
    app.js                - Test harness for simulating app interactions
    i18n.js               - Internationalization utility for tests
    test.public.js        - Main test suite for public app features
    util.js               - Utility functions for tests

How to Run:
-----------
1. Open `public/index.html` in your browser to use the Memo App.
2. The app connects to the API server at https://challenge-server.tracks.run/memoapp by default.
3. Enter a valid UUID v4 access token and click LOGIN to start using the app.

Features:
---------
- Login with access token (UUID v4)
- View memo categories
- Expand/collapse categories to view memos
- Select, edit, save, create, and delete memos
- Responsive design for desktop and mobile

Testing:
--------
- Automated tests are located in the `test/` directory.
- Tests use Mocha, Chai, jsdom, and other utilities.
- To run tests, use Node.js and execute the test files.

Notes:
------
- For development, use files in the `public/` directory.
- The app will use mock data if the API server is unavailable.
- Styling is provided in `public/style.css`.
